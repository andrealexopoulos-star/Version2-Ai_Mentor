# BIQc Foundation And More Features Reclassification

Date: 2026-04-03

## Reclassification Objective
Preserve route continuity while mapping module packaging to `Starter`, `Pro`, `Enterprise`, and `Custom Build`.

## Foundation Position
- `BIQc Foundation` remains the package entry page and maps to tier `Starter`.
- Existing Foundation modules remain launch-ready paid modules with no route renames required.

## More Features Position
- `More Features` remains the staging and demand-capture surface.
- Reclassification target:
  - **Pro graduation set (first):**
    - Risk & Workforce
    - Compliance
    - War Room
    - Analysis
    - Diagnosis
    - Automations
    - Documents Library
    - Market Analysis
    - Ops Advisory Centre
  - **Enterprise graduation set (after Pro stabilization):**
    - Watchtower
    - Data Center
    - Operator Intelligence
    - Intel Centre

## UX/Copy Alignment Rules
- Foundation page copy should describe `Starter` package explicitly while preserving `BIQc Foundation` naming.
- More Features cards must show staged status and target tier intent (`Pro candidate` or `Enterprise candidate`) once promoted.
- Lock reasons must reference current tier and target tier with no ambiguous language.

## Continuity Notes
- Current route paths are preserved to avoid regression.
- LaunchRoute continues to enforce waitlist behavior until modules are promoted by policy update.
- Promotion workflow should include:
  1. route policy update
  2. matrix update
  3. checkpoint ledger entry
  4. release evidence refresh.
