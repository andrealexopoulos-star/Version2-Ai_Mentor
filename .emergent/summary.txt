**Key Technical Concepts**
-   **Supabase-First Architecture:** FastAPI is used only as a thin client for Supabase. All core logic (AI, business rules) resides in Supabase Edge Functions and SQL.
-   **Deterministic SQL Engines:** Python business logic was successfully migrated to a chain of PostgreSQL functions, which are called via RPC from the main Edge Function. This ensures speed, reliability, and consistency.
-   **No Fake Data Discipline:** The entire frontend was audited and refactored to remove any hardcoded, demo, or placeholder data, instead showing explicit data not available states.
-   **Onboarding State Machine:** The  hook orchestrates the complex, multi-step, gated onboarding flow.

-   **intelligence_snapshots:** Caches the main JSON output from the  Edge Function.

-   **contradiction_memory, escalation_memory, decision_pressure, evidence_freshness:** Tables used by the deterministic SQL engines to track cognitive state between runs.
-   **business_profiles:** Stores the core Business DNA scraped and confirmed during onboarding.
