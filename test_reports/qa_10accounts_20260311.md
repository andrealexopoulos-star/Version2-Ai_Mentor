## ISSUES FOUND

### Issue 1 — Tutorial overlay blocks privacy link ⚠️ MINOR
**Severity:** Low  
**Description:** When tutorial overlay is active, the "See BIQc Privacy Commitment" link cannot be clicked (overlay has z-index 9999 blocking all interactions)  
**Expected:** User should dismiss tutorial first (click "Learn later"), then privacy link works  
**Actual behavior:** Expected — tutorial must be dismissed before page interaction  
**Fix needed:** No code fix needed. This is correct UX flow.

### Issue 2 — Session persistence in automated testing ⚠️ TESTING ONLY
**Severity:** Testing only — not a production bug  
**Description:** Browser session persists between automated test accounts, causing login page to not re-render  
**Fix needed:** None — this is Supabase session management working correctly. Use separate incognito windows for manual testing each account.

### Issue 3 — New users (no calibration) go to `/advisor` if localStorage has onboarding flag 🟡 MEDIUM
**Severity:** Medium  
**Description:** If `biqc_onboarding_done` localStorage flag exists from a previous session, the onboarding modal won't show for returning uncalibrated users  
**Expected:** New users without calibration should go to `/calibration` first  
**Fix needed:** Clear localStorage flag on sign-up, not just on modal close
